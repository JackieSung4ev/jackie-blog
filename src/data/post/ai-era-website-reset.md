The old website did its job. It held drafts, tutorials, experiments, and the occasional post that was mostly a note to
future me. But the web changed, and so did the value of an archive.

AI makes generic explanations cheaper. That does not make writing less useful; it makes context more valuable. A post
is worth keeping when it shows a decision, a constraint, a failure, a workflow, or a point of view that a generic answer
would flatten.

This rebuild starts from that filter.

## What stays

- Posts that still answer a real question.
- Notes that show a process I would repeat.
- Project logs that explain why a decision was made.
- References I still reach for.

## What changes

The site becomes less like a warehouse and more like a working library. The home page points to current writing. The
blog remains simple Markdown. The Now page keeps the direction honest.

The goal is not to publish more. The goal is to keep publishing things that can still be used later.
